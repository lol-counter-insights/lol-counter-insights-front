# LoL Counter Insights

## 1. プロジェクト基本情報

### チーム
- M: デザイナー
- F: マネージャー / エンジニア
- I: エンジニア（未定）

### 役割
- **エンジニア**: システム設計、API連携、実装
- **デザイナー**: ユーザーリサーチ(M&S)、UI/UXデザイン、プロトタイプ
- **マネージャー**: 進捗管理、技術選定

## 2. 背景と目的

### なぜ作るのか？
- LoLのピック画面（短時間）で、どのチャンピオンが有利か迷うストレスを解消するため
- 既存の統計サイト（OP.GG等）は情報が多すぎる/ぱっと見でわかりづらい/英語なため、「カウンター確認」に特化したミニマルなツールを目指す

### プロジェクトの目的
- **M**: ポートフォリオに載せられるレベルの「課題解決型デザイン」の実践と成果物作成。プロダクト制作の経験とスキル向上
課題解決型デザイン: なぜこのデザインにしたか？　を、Webアプリの目的、goalから遡って説明できること
- **F**: プロジェクト管理の経験
- **I**: 対外的に公開可能なWebアプリ開発

## 3. ターゲットユーザー

- **想定ユーザー**: アイアン〜シルバー想定。誰が誰のカウンターなのか曖昧(部分的に知っているレベル)
- **利用シーン**: ランク戦のドラフト中。PCの横にスマホを置いている、あるいはサブモニターで表示

- 大前提として、自分たち(M&S)が使っていて使いやすいアプリであること

## 4. 機能要件

### 言語
- 日本語only
- 翻訳機能の器だけ作っておくと発展性があって良いかも。必須要件ではない。できれば

### 画面仕様
- SPA
- 検索画面、結果画面の2つ

#### 検索画面
- チャンピオンアイコン一覧
- 名前検索（通称・各言語名対応、インクリメンタルサーチ）

#### 結果画面
- 有利/不利TOP5表示
- 外部サイトリンク（OP.GG等）

## 5. デザイン要件

- **デザイナーの注力ポイント**:
  - 競合調査（既存サイトの使いにくい点の抽出、その課題を明確化し、デザインに反映）
  - ユーザーフローの設計(検索の絞り込み結果をどう表現するか)
  - Figmaでの高精度なプロトタイプ作成
  - 完成させるのが目的なので、できるだけ最小要件でミニマムに作ること

## 6. 技術スタック

- **Frontend**: React + Vite
- **Backend**: Cloudflare Workers（相性データ定期集計）
- **API**: Riot Games Data Dragon
- **Deployment**: Cloudflare Pages

## 7. ツール

- **進捗管理**: Jira
- **デザイン**: Figma

## 8. 懸念事項・今後の課題

- データの絞り込み条件（全データは取得不可）
- 参照パッチ（最新パッチのみ？過去パッチも？ 一ヶ月分? 何件??）
  - カウンターはパッチで大きく変わらないので、特定の期日・件数が良さそう
- 参照ランク帯（どのランク帯のデータを使うか、ソロQのみ？）
- データの集計の仕方(同じレーンの勝ち負けのみで良いのか？)
- 極端に少ないデータは捨てるべき？

## 9. マイルストーン

- 1. デザインモック/開発モック作成
- 2. 手作業のデータを入れる ← ここまではそう難しくないはず
- 3. データのリアルタイム取得&集計 ← できたら最強
